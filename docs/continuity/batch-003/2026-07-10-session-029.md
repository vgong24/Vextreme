**FAB auto-load (`docs/architecture/17-fab-autoload.md`, public PR #118).** Victor caught
the Step 7 batch using the wrong FAB layer ("there should be a 'spiral one'... we never
duplicate a category") and then named the deeper principle: common chrome should be added by
*one script*, not hand-edited into N pages. `lib/vextreme.js` now auto-loads the full
spiral-FAB set (`vex-fab.js` + `fab-lang.js` + `fab-theme.js` + `fab-map.js`;
`fab-demo.js` excluded as deprecated) for every `shell.js` page, with a `fabWidgets`
per-widget opt-out. Two structural finds along the way: CI's `build-index.yml` silently
reverts hand-edits to generated `specimen-*.html` pages (the fix had to move into
`lib/build-specimens.js`'s template), and `fab-theme.js` unconditionally overwrites
`document.documentElement`'s `data-theme` on mount.

**The regression cycle (public PR #119) — the most important lessons of the session.**
Victor reported three real regressions from the live site after the rollout: phantom-opera's
authored styles overwritten, the terrain map squashed to 720px, and the FAB absent from
production entirely. Root causes, each now structurally guarded:

1. `vextreme.js` blanket-injected `design-system.css` (universal reset + `:root` tokens +
   global `body` rule) after authored pages' own inline styles — and blanket-loaded
   `section-toggle.js`, which attached collapse-on-click listeners to `fourteen-patterns`'
   own `[data-section]` markup. **Fix:** the whole v1 enhancement layer is now gated to
   actual v1 consumers (a `pages.json` template entry or `#arcNavMount`).
2. The FAB-autoload change shipped without bumping the CDN cache version — production kept
   serving `vextreme.js?v=6` from jsDelivr indefinitely. **Fix:** bumped to `?v=7`;
   `tests/41` now fails CI if `shell.js`'s `VEXTREME_VER` and `vextreme.js`'s
   `DEFAULT_CACHE` ever drift.
3. Nine pages' authored widths exceeded the 720px wrap — including two patterns manual
   checks couldn't see: widths routed through arbitrary-named custom properties
   (`--maxw: 1160px`) and viewport-relative layouts (`max-width:60%`). **Fix:** found by
   the new `lib/audit-fab.js` composition auditor, all given `bodyWrap:false`; `tests/42`
   pins the whole `pages/` tree at zero findings.

Also per Victor: `fab-lang.js` now mounts with a single language ("the language even if
just 1, should still be there").

### Mistakes made

- **The rollout's own regressions (above) are the headline mistakes** — each shipped after
  Playwright verification that checked the wrong invariant (element presence at load, not
  authored-style survival, scroll behavior, or CDN cache reality). The durable answer was
  making each class script-perceptible, not promising more care.
- Step 7's first fix used the standalone legacy FAB layer instead of the spiral container —
  caught by Victor from a screenshot, corrected in the same PR cycle.
- Hand-edited generated `specimen-*.html` files; CI reverted the edit within a minute of
  merge. The generator template was the only real place to fix.
- An early Step 6 verification pass showed `nav: false` on all 8 pages — a harness bug (a
  fresh test copy missing the `shell.js` BASE redirect), not a real failure. Diagnosed
  before trusting it, but it burned a cycle and motivated committing a reusable harness.

### Assumptions that need verification

- jsDelivr's `@main` cache (~12h) will pick up `?v=7` and the gated loader — until then the
  live site serves stale behavior; nothing local can verify this.
- No authored page outside the 8 verified was *deliberately* depending on the injected
  design-system tokens (the gate assumes authored pages want their authored styles).
