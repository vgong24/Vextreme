**Victor asked directly whether the pilot needed "self-honing" to auto-update.** Answered yes,

  `docs/screenshots/` for the existing `{slug}-{lang}.png` convention `scripts/screenshot-page.js`

### Session continued - Archives visibility regression closed

Victor caught a real ambiguity left over from the terrain-map pilot: `terrain-map.html` existed in `pages/`, but production Archives did not list it even though the repo culture now expects new HTML surfaces to become perceivable from the first visible maps.

Root cause: `lib/audit-pages.js`'s `SKIP_PAGES` was doing two jobs. It correctly meant "this page intentionally skips the God Script wiring pattern" for generated/live-fetch dashboards like `terrain-map.html`, but `lib/build-archives.js` also passed the same skip list into `discoverOrphanNodes()`, where it effectively meant "do not surface this page in Archives." Registering Terrain Map as a God Script audit skip prevented a false blocker, but also hid it from the generated page registry.

Fix: Archives now keeps the meanings separate. Content-page auto-discovery still excludes `SKIP_PAGES`, but `lib/build-archives.js` also renders a distinct `System Pages` section from live skipped dashboard pages. `terrain-map.html` can therefore remain classified as "not a God Script consumer" while still appearing in `pages/archives.html` as a reachable system page. This matches the translation-chip culture Victor pointed to: incomplete/not-applicable states should be visible and labeled, not hidden.

This session also exposed an operational teamwork lesson: Codex Desktop follows the shared Git checkout/branch as moved by GitHub Desktop, while Claude's CLI-style work can continue with a more isolated branch perception. The earlier Codex entry-point work survived as a GitHub Desktop stash, so the final branch folded both layers together before opening review: first-page entry points (`index.html`, Archives callout, Ecosystem Hub terrain card) plus the Archives system-page regression fix.
