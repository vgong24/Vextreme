# VEXTREME — Continuity Batch 004 (Sessions 031–040)

[← Batch 003](../batch-003/README.md) · [Continuity index](../INDEX.md)

This batch continues the per-session-file continuity form introduced by Batch
003.

## Rules for this directory

- One file per session, named `YYYY-MM-DD-session-0NN.md`.
- Never edit a prior session file. Same-thread continuations append a
  `### Session continued` block at end of the current file.
- Prefer `node lib/append-session-continuation.js` for continuations.
- Use the session template in `../INDEX.md`.
- Current state, open work, and the batch registry live in `../INDEX.md`.
- When Session 040 closes, create `batch-005/`, update the registry, and begin
  Session 041 there.

## Sessions

The date-prefixed file listing is the session index.
