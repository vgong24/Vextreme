### What was built and why

Terrain already carried graph relationships but required the user to remember why
the camera moved. Phase 1 stores semantic commitments in namespaced `history.state`:
View, Lens, semantic level, stage, selected node, and relationship reason. Camera
coordinates remain ephemeral, so wheel and pan noise never flood browser history.
Clear journey advances a generation marker so Back skips stale entries that the
browser platform retains but the user explicitly removed from the visible path.

Search travel to `lib/build-vextreme.js`, then relationship travel through its
Reads field to `widgets/vex-fab.js`, now produces a visible path. Browser Back
restores the build node, Forward restores the FAB node, and Return to origin
returns to the system overview before browser navigation leaves Terrain.

### Assumptions that held

- `history.state` was the smallest robust Phase 1 mechanism; URL hashes would have
  committed a public deep-link syntax before one was designed.
- The existing overlay drawer could carry Journey without reducing map width.
- Code and Content worlds share enough semantic state shape to use one history module.

### Assumptions that need verification

- Future deep-link work may need a versioned URL projection layered over the current
  namespaced state; Phase 1 deliberately does not define that public contract.
- Browser entries already created by the platform cannot be physically deleted by
  script; generation markers skip cleared entries, but this should be rechecked if
  Terrain later adopts a shareable URL/deep-link contract.
