### Decision

Open PR bodies are the live assignment source. A committed active-work file would
be stale immediately after merge; an open-PR claim naturally leaves the live set
when the PR closes. The checked-in registry stores only stable public-safe actor and
environment references plus fail-closed policy.

Each claim names a session instance, numeric epic position, dependencies, exact path
scope, status, and renewal date. `active` and `review` claims exclude overlapping
active paths. `waiting` claims expose dependency order without reserving paths.

### Boundaries

- Actor and instance references are coordination labels, not authenticated identity,
  legal identity, employment attribution, or proof of authorship.
- Claims do not authorize implementation, merge, acceptance, publication, data
  access, providers, or external effects.
- Public claims contain no private SDK paths, implementation details, private roadmap
  state, contacts, credentials, client data, or private relay content.
- Live GitHub unavailability means assignment state is unknown, never that paths are free.
