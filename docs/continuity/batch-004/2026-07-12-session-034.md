### What was built and why

One self-contained charter page following the handoff's architecture (cover/seal
→ preamble → foundations → constitutional principle → ten Articles →
participants → relational covenant → commitments → practice → promises →
prevent/preserve → distinction → living constitution → north-star → colophon).

Deliberate choices beyond the handoff:

1. **Receipts on every Article** — each Article carries a disclosure thread
   (protects / practiced / prevents) whose "practiced" row cites a real,
   public-safe fact from this repo's own history (the 31-of-39 dead-ends
   audit, the write/read split, the td-006 gate holding, the roles index's
   honest "not yet real" marking, the six drift detectors). A charter that
   demands "no truth without source" owes sources for itself — this is the
   session's main flare, and it implements the handoff's "Relational Article
   Threads" interaction idea as accessible `<details>` elements, no JS needed.
2. **The charter wears the institution's own design system** — palette and
   type are family 1 from `styles/design-system.css` (cream/stone/ember,
   Source Serif 4 / IBM Plex Sans / IBM Plex Mono), inlined for
   self-containment with a comment naming the mirror. No new brand invented.
3. **Colophon names the page's own unverified edges** (not confirmed on the
   production domain; English-only; nav reachability incomplete) — Article V
   applied to the page itself.
4. **Fixed light theme, no runtime toggle** — consistent with od-005's closed
   decision (content pages are light; no page has a stated toggle need).
5. **Minimal JS, all progressive**: copy-link-to-section buttons (hidden
   without JS), and beforeprint/afterprint handlers that open all threads so
   the printed charter is complete. Everything works with JS disabled.
