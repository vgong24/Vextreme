How full coverage gets verified

01
Content pipeline
nodes.json + arcs-v2.json → build-index.js. Asserts every slug resolves, arc order matches priority, and adjacency (prev/next) is correct at both ends of an arc.
↓
02
Strings pipeline
source/*.json → strings-compile.js → compiled bundles + manifest. Asserts every key's hash is stable, duplicate keys don't silently overwrite, and language coverage per key matches what's in each bundle.
↓
03
Browser navigation
Given a slug's position in an arc, asserts the first entry has no prev link, the last has no next link, and arcs render in priority order — the properties a broken nav would violate silently.
↓
04
Build output
Reads the generated HTML directly and asserts specific structure exists — the copy button's click handler, the slug popover, the cell grid — the things a template refactor could silently delete.
