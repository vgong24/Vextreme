Localization Source-Truth Demo

Where a string lives, and whether it can be trusted.

A static, public-safe look at the private Vextreme SDK's localization source-of-truth product — the same private implementation whose identity resolver is shown on the SDK Identity Demo page, here shown working across a small body of content rather than one call.

What this demo shows

The private Vextreme SDK maintains a registry of canonical UI strings — one record per piece of translatable content, tracking where it lives, what language values it has, and whether anything about it needs attention. Two things the registry can answer that a plain translation file usually can't: is this content correctly identified, and is anything missing.

Misalignment finding

An identity signal recorded under one element's usage actually resolves to a different element — the kind of thing that would otherwise sit silently wrong until someone happened to notice:

Finding

Recorded under Hero subtitle
Actually belongs to Hero title
Confidence strong
Why A shared naming pattern pointed to the title element, not the subtitle element it was recorded under.

This is flagged for review, not silently trusted and not silently corrected — a human decides what to do with a finding like this.

Evidence and source-truth relationship

The same registry can connect a piece of content back to visual evidence — a screenshot showing where and how it actually appears — and forward to every other place it's used. This page doesn't demonstrate that traversal directly; it exists in the private product as a way to move from "here's a screenshot" to "here's the exact content and status behind it," and back.

Vendor round-trip principle

When translated content comes back from outside the system — a vendor, a translator, a reviewer — it's compared against what was actually sent out before anything changes. Real, intentional edits are proposed for review. Anything that looks like it touched a field it shouldn't have, or arrived stale relative to a newer change already made to the source, is flagged instead of applied. Nothing is ever silently overwritten.

This is a static public demonstration artifact. The private Vextreme-SDK generated this sanitized output from a synthetic registry, and public Vextreme displays the result. No live connection exists between this page and the private SDK — there is no upload, no live call, no backend, and nothing on this page can trigger the private system to do anything. Viewing this page does not grant access to the SDK, private code, secrets, or client data.

What this demo does not show

  • No private SDK source code.
  • No client data — every value on this page is invented.
  • No secrets, credentials, or tokens.
  • No live API — this page makes no network calls to anything private.
  • No authenticated user flow, no file upload, no "analyze your own content" interaction.
  • No production service — this is a controlled proof, not a live product.
  • No screenshot evidence viewer, no vendor packet format, no import tooling.
  • No full localization engine — this shows a slice of what the product finds, not the whole system.
