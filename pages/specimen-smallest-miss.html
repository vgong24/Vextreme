How a one-line change gets caught

01
enHash recorded at compile time
Every key's English text is hashed into manifest.json when it compiles cleanly — a 12-character fingerprint of exactly what a translator translated against.
↓
02
English text changes
A future edit — even a single word — changes what the English source hashes to. Nothing about this requires the edit to be large or obviously risky.
↓
03
strings-check.js runs before every compile
It recomputes the hash and compares it to manifest.json. A mismatch means the English changed since this key's translations were written — that's a WARN, one of five severity levels (BLOCK, REMAP, WARN, INFO, quarantine), not a hard failure.
↓
04
_stale: true, written back automatically
The translation isn't blocked from shipping — it's tagged. The badge above this text on the Japanese version of this page is that tag, produced by a real run of this check against this exact key, not written in by hand.
